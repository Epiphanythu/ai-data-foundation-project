```

这个问题背后的含义是：
同样收入和信用评分的人，在不同地区违约风险相同吗？
不同经济周期下，比如2008金融危机vs2015年，风险有何差异？

传统模型只看个人特征，忽略了地区经济和宏观周期。
```

```
第二，回答地区经济和宏观环境是否独立影响个人违约风险这一学术问题。

帮助金融机构降低坏账损失，
```

**【幻灯片11：方法一 - 流式处理】**

```

方法一：流式处理解决大规模数据加载问题。

（指向红色区域 - 问题）
问题是：2.8GB CSV文件一次性读取导致内存溢出。

（指向蓝色区域 - 解决方案）
我们的解决方案是：
使用 pandas 的 read_csv 方法，设置 chunksize=10000，
每次只读1万条记录，处理完成后丢弃，再读下一万条。

（指向绿色区域 - 结果）
结果是：内存占用始终保持在低水平。

这个方法让我们能够顺利处理226万条记录。
```

**【幻灯片12：方法二 - 违约标签构造】**

```

方法二：严格违约标签构造确保结论可靠性。

（指向问题）
问题是：贷款状态有10多种，Current（正在还款）将来是否违约未知。

（指向解决方案）
我们的决策是：
只保留已完结状态：Fully Paid（全额还款）、Charged Off（核销）、Default（违约）。

（指向结果）
这个决策让我们从226万条记录中筛选出136万条已完结样本。
虽然样本量减少了，但标签质量大大提高了。
```

**【幻灯片13：方法三 - 时间对齐】**

```

方法三：时间对齐匹配贷款发放时的宏观环境。

（指向问题）
问题是：贷款数据跨越2007到2018年，FRED数据为月度频率。

（指向解决方案）
我们的方法是：
FRED月度数据取季度最后一个月值，与贷款发放季度对齐。

（指向结果）
这样做的结果是：
匹配发放时的宏观环境，避免使用未来数据。

这一点很重要，因为在实际应用中，我们只能用发放时的信息来预测违约。
```

**【幻灯片14：方法四 - 残差分析】**

```

方法四：残差分析证明地区经济的独立解释力。

这个方法比较复杂，让我详细解释一下。

（指向问题）
我们观察到：贫困率高的州违约率也高。
但有人说：这是因为贫困州高风险贷款更多吧？
比如这些州的人信用分本来就低，或者贷款利率本来就高。

（指向解决方案）
为了回答这个问题，我们用了残差分析方法：
第一步，建立利率到违约率的模型。
第二步，用这个模型预测每个州的"预期违约率"。
第三步，计算残差等于实际违约率减去预期违约率。
第四步，看残差和贫困率是否还相关。

（指向结果）
结果是：
即使控制了利率，相关性从0.398降到0.362，仍然显著。
这证明了地区经济有独立于个人特征的解释力。
```

**【幻灯片17：发现二 - 利率相关】**

```

发现二：利率与违约高度相关，反映平台风险定价能力。

（指向图表左侧）
低利率，小于8%的贷款，违约率只有6.38%。
高利率，大于等于24%的贷款，违约率达到48.56%。

（指向右侧发现）
违约率随利率上升几乎呈线性增长。

```

**【幻灯片18：发现三 - 宏观环境】**

```

发现三：宏观金融环境与违约率季度相关系数达0.844。

（指向图表左侧）
蓝色线是违约率，红色线是联邦基金利率。
大家可以看到，两条线的趋势非常相似。

（指向右侧发现）
我们计算了相关系数：0.844。
这是一个很高的相关性。

但我要诚实说明：
此为相关性分析，不能简单解释为因果关系。
可能是利率上升导致违约率上升，
也可能是经济不好导致两者同时上升。

不过，这个发现仍然很有价值：
它说明宏观环境值得纳入风险分析框架。
```

**【幻灯片19：发现四 - 地区经济】**

```

发现四：地区经济具有独立于个人特征的解释力。

这是我们最重要的发现之一。

（指向图表左侧）
州级贫困率与违约率的相关系数是0.398。

（指向右侧发现）
但是，有人可能会质疑：
是因为贫困州高风险贷款更多吧？

所以我们控制了州级的平均利率，重新计算相关性。
结果是：0.362，仍然显著。

我们还控制了州级的平均FICO信用分。
相关性反而上升到了0.459。

这说明什么？
地区经济有独立于个人特征的解释力。
即使两个人信用评分相同、利率相同，
生活在贫困率更高的州，违约风险仍然更高。
```

```
（指向图表左侧）

G等级加上debt_consolidation（债务整合），

（指向右侧发现）
这意味着什么？
每两笔这样的贷款，就有一笔会违约。

这个发现有实际应用价值：
风控部门可以直接识别这些高风险组合，
```

## Q&A 准备

### 可能被问到的问题

**数据处理类：**
- "226 万条数据是如何处理的？"
  → 流式读取，chunksize=10000，逐块处理，避免内存溢出

- "州级数据对齐的具体逻辑是什么？"
  → 按 addr_state 分组，计算违约率/均值，与 ERS 数据 join

- "控制变量分析是怎么实现的？"
  → 残差分析法：计算实际违约率减去预测违约率，分析残差与贫困率相关

**方法论类：**
- "为什么不用机器学习模型？"
  → 中期重点是数据基础和可解释分析，模型是下一阶段

- "相关性≠因果，你的结论有什么意义？"
  → 相关性说明变量值得纳入，下一阶段会做因果推断

- "Lending Club 能代表所有贷款吗？"
  → 不能完全代表，但数据规模大、字段丰富，适合课程项目

**项目管理类：**
- "两个人是如何分工的？"
  → A 负责数据获取和清洗，B 负责分析和建模

- "如果时间不够，哪些部分可以砍掉？"
  → 保多源分析，模型可选

## 演讲技巧提示

1. **开场要有吸引力**
   - 用问题开场，引发思考
   - 语速适中，不要太快

2. **讲解图表时**
   - 先指出整体趋势
   - 再指出关键数据点
   - 最后说明含义

3. **诚实面对限制**
   - 相关性不是因果
   - 数据不是完全代表性
   - 模型是下一阶段工作

4. **时间控制**
   - 每部分严格控制时间
   - 如果超时，精简例子，保留核心结论

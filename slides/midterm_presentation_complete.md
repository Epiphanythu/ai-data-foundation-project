## 1. 研究动机（2分）

### 1.1 为什么这是一个重要的项目？

**实际价值**：
- 个人贷款违约直接影响金融机构坏账损失
- 影响借款人的信用定价和融资成本
- 风险定价不准会导致不公平现象
- 多源数据可以提升定价精准度，具有商业应用价值

**科学意义**：
- 传统模型只看个人特征，忽略了环境和周期因素
- 回答"地区经济和宏观环境是否影响个人违约风险"这一学术问题
- 验证三层风险模型：违约风险 = 个人特征 + 地区经济 + 宏观周期

### 1.2 为什么具有挑战性？

**挑战一：数据规模大**
- Lending Club accepted.csv 约 2.8GB
- 226万条贷款记录，151个字段
- 一次性加载会导致内存溢出

**挑战二：标签复杂**
- 贷款状态有10多种：Fully Paid, Charged Off, Current, Late, In Payment Plan...
- 未完结状态（Current, Late）无法判断最终是否违约
- 错误的标签定义会导致研究结论偏差

**挑战三：多源数据粒度不一致**
- 贷款数据：个人级别 + 发放时间
- 州级数据：50个州的年度/季度平均值
- 宏观数据：国家层面的月度/季度时间序列
- 如何将三个粒度的数据有效融合？

## 2. 相关工作（2分）

### 2.1 现有相关工作

| 方法类型 | 代表技术 | 优点 | 局限 |
|----------|----------|------|------|
| **传统信用评分** | FICO、收入、负债率 | 解释性强，易于理解 | 只看个人，忽略环境和周期 |
| **机器学习预测** | LR、RF、XGBoost、神经网络 | 预测能力强，准确率高 | 容易黑盒化，缺乏可解释性 |
| **多源风险分析** | 地区收入、失业率、利率 | 视角全面，贴近实际 | 数据对齐困难，口径不一致 |

### 2.2 它们的局限性是什么？

**传统方法的局限**：
- 假设同样收入、同样信用评分的人，违约风险相同
- 忽略了地区经济差异（如密西西比州 vs 加州）
- 忽略了宏观周期差异（如2008年金融危机 vs 2015年）

**机器学习的局限**：
- 虽然预测准确，但无法解释"为什么这个地区风险高"
- 黑盒模型难以被风控部门接受和采用

**多源分析的局限**：
- 数据对齐是技术瓶颈
- 缺乏控制变量分析，难以区分相关与因果

### 2.3 为什么需要提出新的解决方案？

**本研究定位**：
- 不是取代传统方法，而是补充外部维度
- 先做好数据清洗、融合、可解释统计分析
- 再考虑预测模型
- 强调多源数据的增量价值和可解释性

#### 方法论讲解（5分钟，同学 B）

```
谢谢同学A。

在介绍具体发现之前，我想先讲一下我们的分析方法。
因为同样的一份数据，用不同的处理方法，可能得到完全不同的结论。

【方法1：违约标签构造】
首先，最基础的问题是：什么是违约？

Lending Club 的原始数据有10多种贷款状态。
如果我们把"Current"（正在还款）当作不违约，
那就会低估真实的违约率——这些贷款将来可能还是会违约。

所以我们的决策是：只保留已完结状态。
具体就是 Fully Paid（全额还款）、Charged Off（核销）、Default（违约）。

这个决策让我们从226万条记录中筛选出136万条已完结样本。
虽然样本量减少了，但标签质量大大提高了。

【方法2：流式处理】
第二个技术问题是数据规模。
2.8GB 的 CSV 文件，一次性读取会内存溢出。

我们用的方法是分块处理：
每次只读 1 万条记录，处理完后丢弃，再读下一万条。
这样内存占用始终保持在很低的水平。

【方法3：时间对齐】
第三个问题是时间对齐。

我们的贷款数据跨越 2007 到 2018 年，共 47 个季度。
FRED 的宏观数据是月度的，我们取每季度最后一个月的值作为代表。

为什么要这样做？因为贷款是在季度内发放的，
我们希望匹配发放时的宏观环境，而不是用未来的数据。

【方法4：控制变量分析】
第四个，也是最重要的方法，是控制变量分析。

当我们发现"贫困率高的州违约率也高"时，
有人说：这其实是因为贫困州的人信用分本来就低，或者贷款利率本来就高。

为了回答这个问题，我们用了残差分析方法：
第一步，建立利率→违约率的模型
第二步，预测每个州的"预期违约率"
第三步，计算残差 = 实际 - 预期
第四步，看残差和贫困率是否还相关

结果是：即使控制了利率，相关系数从 0.398 降到 0.362，仍然显著。
这证明了地区经济有独立的解释力。

有了这套分析方法，我们才敢说下面的发现是可靠的。
```

```
地区经济有独立于个人特征的解释力。
即使两个人信用评分相同、利率相同，
生活在贫困率更高的州，违约风险仍然更高。

【发现5：组合风险】
最后一个发现：组合风险分层能识别高风险人群。

【指向图表】
如果我们把等级和贷款用途组合起来看：
G等级 + debt_consolidation（债务整合）
违约率高达 52.99%

这意味着什么？每两笔这样的贷款，就有一笔会违约。

这个发现有实际应用价值：
风控部门可以直接识别这些高风险组合，
进行更严格的审核或定价。
```

## 6. Q&A 准备

### 可能被问到的问题

**数据处理类**：
- "226 万条数据是如何处理的？"
  → 流式读取，chunksize=10000，逐块处理，避免内存溢出
  - 相关代码：`analyze_lending_club.py`

- "州级数据对齐的具体逻辑是什么？"
  → 按 addr_state 分组，计算违约率/均值，与 ERS 数据 join
  - 相关代码：`build_ers_state_features.py`

- "控制变量分析是怎么实现的？"
  → 计算残差 = 实际违约率 - 预测违约率，分析残差与贫困率相关
  - 相关代码：`build_state_control_analysis.py`

**方法论类**：
- "为什么不用机器学习模型？"
  → 中期重点是数据基础和可解释分析，模型是下一阶段

- "相关性≠因果，你的结论有什么意义？"
  → 相关性说明变量值得纳入，下一阶段会做因果推断

- "Lending Club 能代表所有贷款吗？"
  → 不能完全代表，但数据规模大、字段丰富，适合课程项目

**项目管理类**：
- "两个人是如何分工的？"
  → A 负责数据获取和清洗，B 负责分析和建模

- "如果时间不够，哪些部分可以砍掉？"
  → 保多源分析，模型可选
